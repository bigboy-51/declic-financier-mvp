Implement login flow with couple/single detection and profile selection:

1. **On User Login - Initial Check**
   
   After Firebase authentication success:
   - Get currentUser.uid
   - Query Firebase: users/{uid}/userType
   
   If userType doesn't exist (new account):
   - Show modal: "Seul(e) ou En couple?"
   - Options: [Seul(e)] [En couple]
   - On selection: Save users/{uid}/userType = "single" or "couple"
   - Proceed to Step 2
   
   If userType exists:
   - Proceed to Step 2 (Profile selection if couple)

2. **Profile Selection Modal (Couple Mode Only)**
   
   If userType = "couple":
   - Show modal: "Qui êtes-vous?"
   - Options: 
     * [Je suis le mari / Chef de famille A]
     * [Je suis l'épouse / Chef de famille B]
   - On selection:
     * Save to localStorage: currentProfile = "boss_profile" OR "wife_profile"
     * Proceed to Step 3
   
   If userType = "single":
   - Skip profile selection
   - Set localStorage: currentProfile = "single_profile"
   - Proceed to Step 3

3. **localStorage Management - currentProfile**
   
   Key: "currentProfile"
   Values: "boss_profile" | "wife_profile" | "single_profile"
   
   Lifecycle:
   - Set on profile selection (couple) or after login (single)
   - Persist across pages/refreshes
   - Used in ALL Firebase queries: users/{uid}/[currentProfile]/
   - Clear on logout: localStorage.removeItem("currentProfile")
   
   Implementation:// On profile selection
localStorage.setItem("currentProfile", "boss_profile");// On any Firebase query

5. **Navigation Flow**
   
   NEW ACCOUNT (First login):
   Login → Check userType (doesn't exist)
   → Modal: "Seul(e) ou En couple?"
   → If "Seul(e)": 
      * Save userType = "single"
      * Set localStorage: currentProfile = "single_profile"
      * Proceed to PROMPT 4A-2
   → If "En couple": 
      * Save userType = "couple"
      * Show modal: "Qui êtes-vous?"
      * Set localStorage: currentProfile = "boss_profile" OR "wife_profile"
      * Proceed to PROMPT 4A-2
   
   RETURNING ACCOUNT - Single (Login 2+):
   Login → Check userType = "single"
   → Skip profile selection
   → Set localStorage: currentProfile = "single_profile"
   → Proceed to PROMPT 4A-2
   
   RETURNING ACCOUNT - Couple (Login 2+):
   Login → Check userType = "couple"
   → Show modal: "Qui êtes-vous?"
   → On selection: Set localStorage: currentProfile = "boss_profile" OR "wife_profile"
   → Proceed to PROMPT 4A-2

6. **Edge Cases & Error Handling**
   
   Case 1: User closes profile selection modal
   - Modal stays open (required selection)
   - Cannot proceed without selecting
   
   Case 2: User changes device/browser
   - localStorage is per-device
   - Profile selection modal appears again (normal)
   - Select profile → All data loads for that profile
   
   Case 3: localStorage cleared by browser
   - On next action: Redirect to profile selection
   - Re-select profile
   - Continue
   
   Case 4: User logs out
   - Clear localStorage: currentProfile
   - On next login: Show profile selection again
