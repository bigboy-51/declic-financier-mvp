Setup Firebase structure for Charges with 7 fixed categories:

2. **Firebase Data Structure**
   
   Charges stored at:
   users/{uid}/charges/
   
   Structure:
users/{uid}/charges/
├── {categoryId}/
│   ├── name: string (e.g., "Finances")
│   ├── icon: string (e.g., "💰")
│   ├── order: number (1-7)
│   ├── locked: boolean (true - cannot edit/delete)
│   ├── createdAt: timestamp
│   └── rubriques/
│       └── {rubriqueId}/
│           ├── name: string
│           ├── prevu: number
│           ├── reel: number
│           ├── createdAt: timestamp
│           └── updatedAt: timestamp
   
   Example:
users/{uid}/charges/finances/
├── name: "Finances"
├── icon: "💰"
├── locked: true
└── rubriques/
└── impot-001/
├── name: "Impôts"
├── prevu: 200
├── reel: 200

3. **Initialize Categories on First Login**
   
   In FirebaseDataContext.tsx (or AuthContext.tsx):
   
   On successful login:
   - Check: users/{uid}/charges/ exists?
   - If NO (new user):
     * Create all 7 categories automatically
     * Initialize each category with: name, icon, order, locked: true
     * Initialize rubriques: [] (empty)
   - If YES (returning user):
     * Skip initialization (already exists)
   
   Function to create:
```typescript
   async function initializeChargesForNewUser(uid: string) {
     const chargesRef = ref(database, `users/${uid}/charges`);
     const chargesSnapshot = await get(chargesRef);
     
     if (!chargesSnapshot.exists()) {
       // New user: Create all 7 categories
       for (const category of CHARGE_CATEGORIES) {
         await set(ref(database, `users/${uid}/charges/${category.id}`), {
           name: category.name,
           icon: category.icon,
           order: category.order,
           locked: true,
           createdAt: new Date().toISOString()
         });
       }
     }
   }
```
   
   Call this function:
   - After user login successful
   - Pass currentUser.uid

5. **Firebase Security Rules**
   
   Update rules to protect categories:
   
```json
   {
     "rules": {
       "users": {
         "$uid": {
           ".read": "auth.uid === $uid",
           ".write": "auth.uid === $uid",
           "charges": {
             "$categoryId": {
               ".write": "!data.child('locked').val()" 
             }
           }
         }
       }
     }
   }
```
   
   This prevents deletion of locked categories (backend validation)
